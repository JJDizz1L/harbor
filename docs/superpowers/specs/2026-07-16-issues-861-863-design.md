# Issues 861 and 863 Fix Design

## Goal

Prevent MAL movie playback from crashing when no episode exists, and prevent native HTTP startup bursts from queueing unrelated UI data for minutes.

## Issue 861: MAL movie stream identifiers

`buildStreamIds` currently checks anime episode alignment before it knows an episode exists. A MAL movie such as `mal:32281` has no episode, so evaluating `episode.episode` throws before the play picker can build movie identifiers.

The alignment and mapped IMDb episode identifier will be computed only inside an `episode` guard. Movie identifiers will continue through the existing movie fallbacks, producing the MAL identifier and any resolved IMDb movie identifier without inventing season or episode fields.

## Issue 863: native HTTP startup starvation

The shared native HTTP client limits all platforms to ten concurrent requests. Each request may occupy a permit for thirty seconds, but time spent waiting for a permit is outside the request timeout. A startup burst of roughly one hundred requests can therefore wait through ten waves, or about five minutes. This is especially harmful on Windows, although the limit was introduced to protect Linux `systemd-resolved`.

The concurrency guard will apply only on Linux. Every native fetch on every platform will also receive one absolute deadline that begins before permit acquisition and covers permit waiting, request sending, and response body reading. The existing `timeoutMs` argument remains the source of the deadline, with thirty seconds as the default. The shared client and Hickory DNS configuration remain unchanged.

## Error handling

A native request that exceeds its absolute deadline returns a descriptive timeout error. Existing frontend callers already handle rejected fetches and retry or degrade to empty data as appropriate. Permit acquisition failures retain their current semaphore error.

## Testing

- A TypeScript regression test will call `buildStreamIds` with a MAL movie, no episode, and a resolved IMDb identifier. It must return both movie identifiers without throwing.
- Rust unit tests will verify that the concurrency guard is enabled only for Linux and that an absolute deadline includes time spent waiting for a saturated semaphore.
- Project verification will run the focused tests, the full JavaScript test suite, `vp check` on changed files, `vp run typecheck`, and `cargo check --manifest-path src-tauri/Cargo.toml` before the branch is pushed and the pull request is created.

## Scope

The fix does not change stream ranking, addon ordering, Linux's DNS protection, or startup data-loading architecture. It does not alter unrelated work in the user's original checkout.
